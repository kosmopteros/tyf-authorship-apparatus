# TYF Foundation

TYF is not a writing assistant. TYF is not a productivity system. TYF is not a knowledge-management tool.

**TYF / The Yours Faithfully** is a faithful apparatus for authorship. Its job is to preserve source, elicit knowledge, protect register, propose edits, and enforce the controlled write. It does not generate a whole book from a single prompt. It helps a person write a complete body of work on their topic, in their voice.

It is not a ghostwriter, not a knowledge-management tool, not a productivity app, and not a content engine. Every skill serves one of five roles, and anything that serves none of them is out.

## What sits where

The doctrine above is the center. Everything below serves it, in this order of permanence:

```
TYF skills          the core: the disciplines themselves
tyf helper          enforcement: the single writer into manuscript/
workspace contract  state: plain, inspectable files
Cowork              one strong runtime
other harnesses     portability targets
```

A runtime is a place TYF runs, never part of its doctrine. Cowork is the first strong runtime; the skills, the helper, and the contract outlive it.

## The five roles

1. Interviewer
2. Amanuensis
3. First reader
4. Faithful editor
5. Redactor (typographer)

## One vocabulary

TYF uses one plain vocabulary. A skill is named for what it does, and that name is the same in the file path, the frontmatter, the docs, and the prose. There is no second ceremonial surface to learn. Each pass maps to a skill:

| Pass | Skill |
|---|---|
| intake of raw material | `ingesting-sources` |
| interview for tacit knowledge | `interviewing-the-author` |
| structure knowledge | `structuring-knowledge` |
| manage voice and registers | `managing-voice` |
| compose drafts (under write control) | `composing-as-amanuensis` |
| sympathetic read | `reading-sympathetically` |
| diagnose | `diagnosing-text` |
| propose and controlled revise | `editing-faithfully` |
| adversarial audit | `auditing-adversarially` |
| the only path into the manuscript | `controlling-manuscript-writes` |

## The core idea: the Zoom x Pass matrix

Everything the editorial apparatus does is one operation across two axes.

**Zoom (how much text is in scope):** Argument, Architecture, Section, Paragraph, Sentence, Glyph. The top three are knowledge bands, where the author's expertise is irreplaceable. The bottom three are craft bands.

**Pass (what the operation does):** Elicit, Read, Diagnose, Propose, Compose, Revise, Audit. They group into three stances: eliciting, read-only reporting (Read, Diagnose, Audit), and controlled writing (Compose and Revise, behind the controlled write).

A skill is roughly one `(Band x Pass)` cell. Elicit concentrates in the knowledge bands; Compose in the middle bands; Revise in the craft bands. Read, Diagnose, and Audit are three different stances toward the same text and none replaces the others.

## The redactor spans all bands (Milchin)

Integrity is a substrate, parallel to voice. Voice registers govern how the work sounds; the redactor canon governs whether it holds together. The Milchin (редактор) tradition is structural and logical, not glyph-only, so the same integrity discipline runs at every magnification: micro (glyph, sentence), macro (section, architecture, argument), and meta (the whole work and its running style sheet). The redactor is not a band. It threads through the diagnosis pass in Diagnose, the editor in Propose and controlled Revise, and the adversarial audit in Audit. See `keeping-the-redactor-canon`.

## The lifecycle layer

Around the editorial apparatus sits the workspace lifecycle: `initializing-a-workspace` founds a workspace and runs intake, `working-the-workspace` governs where each pass may read and write, `scheduling-ongoing-work` runs the iterative-phase cadence (on Cowork, via scheduled tasks and Routines), and `keeping-documentation-honest` makes sure no structural change is called done while the docs that route future behavior are still stale. The `tyf` helper performs the concrete file operations and is the single writer into `manuscript/`.

## Note on the knowledge bands

The three knowledge-band disciplines the manifesto would promote to their own cells (thesis interrogation, argument spine, claims index) currently live inside `interviewing-the-author`, `structuring-knowledge`, and `auditing-adversarially`. Promoting them remains later roadmap work and needs no restructuring.
